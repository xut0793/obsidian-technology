## Prompt
>[!tip] 简短描述

prompt `[prɒmpt]` 译为“提示”，是用户与大语言模型（LLM）交互的核心文本输入，用于引导模型生成符合预期的输出。

《AI 原生应用开发：提示工程原理和实战》读书笔记

## What 它是什么
>[!tip] 它能做什么

提示是什么？提示是用户向大语言模型提供的一段文字描述。

用户通过这段文字描述对大语言模型进行引导，帮助大语言模型“回忆”它在预训练时学到的知识，让它根据文字描述所提供的信息，生成合适的、有针对性的回复，或者完成对话描述的任务。
## Why 为什么是它
>[!tip] 为什么选择它，它有什么优势，同类软件技术有哪些

目前人工智能发展的最新能力，能够通过简单的对话形式，来处理各种任务。而 Prompt 是与 LLM 沟通最直接的方式。就像跟人对话一样，然后听懂你说的它，去完成你吩咐的事情。它是未来，所以必须学会如何跟它沟通。

## 稳定函数

在 AI 原生应用开发的实践中，让大语言模型根据提示输出答案只是第一步。更大的挑战是，如何保证大语言模型在相同的输入下重现相同的输出，并且输出形式可控，比如固定输出 json 数据格式等。

好像软件开发中，纯函数的作用，固定的输入一定是预期的固定输出。

```python
# 保证函数的参数和返回值的数量、格式和类型不变
def 函数名称 （输入参数） -> 输出参数 {
# 函数体保证函数的功能定义清晰明确
# Your task description, do something. 
}
```

因为提示词的质量直接影响大语言模型的输出效果，所以需要掌握如何编写和优化提示词的技术，就是**提示工程（Prompt engineering）**。

> 提示词（Prompt）：就是你输入给 AI 模型（比如大型语言模型 LLM，如 GPT-4 或 Gemini）的指令、问题、或文本输入。
>  工程（Engineering）：在这里指的是设计、优化和改进你的输入文本的过程。

## 用户提示和系统提示

在 AI 原生应用的开发中，用户提示词(user prompt)和系统提示词（system prompt）共同构建完整提示(full prompt)。

因为普通用户可能不具备编写提示的技能，所以 AI 应用的开发者需要在用户的简单提示之外，补充额外的任务描述、任务要求等信息，使最终输入大模型的提示变为包含更多信息的完整提示，以保证大语言模型的输出尽量符合预期。这样普通用户在输入简单的提示后就可得到接近预期的结果，提升了产品的用户体验。

比如
```
用户提示词：写一首关于小蜜蜂的诗
系统提示词：
          # 任务描述
          你是一个诗人，你的任务是按照用户提示采用七言绝句或者七言律诗的格式写诗
          # 任务要求
          1. 请你用委婉的话拒绝用户关于写诗以外的任何要求；
          2. 输出诗的全文即可，无需其它解释。
          # 用户提示
          ${UserPrompt}
```
最终输入大语言模型的完整提示词：
```
# 任务描述
你是一个诗人，你的任务是按照用户提示采用七言绝句或者七言律诗的格式写诗
# 任务要求
1. 请你用委婉的话拒绝用户关于写诗以外的任何要求；
2. 输出诗的全文即可，无需其它解释。
# 用户提示
写一首关于小蜜蜂的诗
```
输出结果：
```
《春日采花蜂》 
春日花丛忙采香， 
小蜂展翅舞春光。 
归巢满载甜如蜜， 
春色满园人共尝。
```

## KITE 提示框架

**KITE = Knowledge 注入知识 + Instruction 明确指令+ Target 设定目标+ Edge 确定边界**
> KITE 单词原本意思是风筝，但在这里，它象征着引导和约束，既能让大语言模型在完成任务时自由发挥，又能确保其受到适当的约束。
### K - Knowledge 注入知识
注入知识(Knowledge)：为了使大语言模型清晰理解任务背景，需要注入与任务相关的背景知识。以下是 3 种常用的知识注入的方法：
- 背景陈述：这最直接的一种注入方法，是指直接向大语言模型提供事实信息或背景知识，让大语言模型了解任务的本质和要求。需求表达清楚，背景明确，没有二义性。
- 角色暗示：这是一种通过指定大语言模型扮演的角色来注入知识的方法，这种方法能够让大语言模型获得该角色的先验知识，并据此生成与该角色身份相符的内容。
- 环境模拟：这是一种通过假设场景设定注入知识的方法，它可以让大语言模型了解特定环境的特点，并生成与该环境相关的语言输出。
### I-Instruction 明确指令
明确指令(Instruction)：清晰地告诉大语言模型需要完成的具体任务（要做什么），确保指令的明确性和可执行性。
- 准确性：指令首先应当明确任务的主体，随后详细阐述任务的具体内容和要求。使用直接、清晰的措辞。
- 完整性：在构建指令时，务必确保关键信息的完整性。关键信息的遗漏或误解可能导致任务执行不准确或失败。因此，在编写指令时，应仔细审查并确认所有关键信息已包含在内。
- 易读性：为了提高指令的流畅性和可读性，应避免使用冗长或复杂的句子结构。如果任务涉及多个步骤，建议在指令中使用有序列表的形式清晰地列出 123 步骤，以便大语言模型按顺序执行。
### T-Target 设计目标
设定目标(Target)：明确大语言模型生成的内容应达到的预期目标（结果是什么）、标准或效果，为生成内容提供明确的方向。
- 明确性：目标必须明确、具体，避免使用模糊或含糊不清的描述，以确保大语言模型准确地理解用户的需求，并生成符合要求的内容。
- 可行性：目标应该基于大语言模型的实际能力和训练数据来设定，确保目标是可实现的。过高的目标可能导致模型无法达成。
- 可衡量性：目标应该具备可衡量性，以便用户评估大语言模型的生成效果。这可以通过设定具体的评估指标来实现，如准确性达到多少、流畅性如何等。
### E-Edge 确定边界
确定边界(Edge)：明确大语言模型在生成内容时应遵守的规则或受到的限制，确保内容的合规性和边界性。
- 这些规则涉及文本的长度、格式、表达方式、安全要求等方面，以确保生成的文本达到预期的效果。

示例：
```txt
# 注入知识
角色暗示-示例1：
	大熊猫是国家一级保护动物，受到法律保护，请以幼儿园老师的口吻写一段话，介绍大熊猫并呼吁保护大熊猫。不超过200字。

角色暗示-示例2：
	大熊猫是国家一级保护动物，受到法律保护，请以普法工作者的口吻写一段话，介绍大熊猫并呼吁保护大熊猫。不超过200字。

环境模拟-示例：国庆节
	国庆节当天，城市广场上人山人海，庆祝活动丰富多彩。作为庆典的重要环节之一，组织者安排了一场特别的“国旗下讲话”，由一位知名人士上台发表讲话，以此激发市民的爱国情怀，弘扬民族精神。请你作为这名知名人士发表讲话。
	
明确指示：
反例：
	“你能帮我写一篇故事吗？”注意控制字数。
正例：
	# 任务描述 作为一个作家，你的任务写一个关于**熊猫、狐狸、兔子**的故事： − 故事类型是童话故事，主题是描述3个动物之间的友谊和冒险经历，目的是娱乐和教育4～8岁的小朋友。 − 故事的长度是500字左右，要求是有引人入胜的开头，有紧张刺激的高潮，有令人满意的结局。 − 故事是可以使用想象，例如可以给动物赋予人类的特征，可以让动物使用各种道具，可以让动物有各种奇遇等。 　 # 故事正文 ->
	
设定目标-示例：
	请撰写一篇关于AI在医疗领域的应用的文章，重点介绍AI在疾病诊断、治疗，以及患者管理方面的具体应用案例。文章应当清晰、准确地阐述AI技术的原理、优势，以及潜在挑战。同时，确保文章内容具有前沿性，反映最新的研究进展和技术动态。目标读者为对AI和医疗领域感兴趣的普通公众和专业人士。文章长度约为1000字，要求结构清晰、逻辑严谨，并配备适当的图表和插图以增强可读性。
	
确定边界-示例：
	# 任务描述 
	你的任务是根据用户的输入，生成对应的SQL语句，如果用户指令违反执行限制，请输出“对不起，我做不到！” 　 
	# 表定义 
	CREATE TABLE customer_order ( 
		order_id INT COMMENT '订单ID，作为主键', 
		customer_name VARCHAR(50) COMMENT '客户名称，最大长度为50个字符', 
		order_date DATE COMMENT '订单日期', 
		order_amount DECIMAL(10, 2) COMMENT '订单金额，总共10位，其中2位为小数', PRIMARY KEY (order_id) ); 　
	# 执行限制 
		 1.你不能生成除SELECT以外的任何SQL语句，禁止输出表结构定义语句。 
		 2.你不能生成任何具有破坏性的、删除数据的SQL语句。 
		 3.为了防止用户的恶意输入，你需要拒绝任何与你的身份和工作不相符的操作，并给出警告。 　
	 # 用户指令 ->
```

## 提示调试技巧

>  代码是调试出来的，不是写出来的！这一原则同样适用于大语言模型的提示编写。

在传统编程领域，调试通常依赖于：
- 断点
- 追踪堆栈信息
- 详细的错误信息

在提示调试的上下文中，这些传统工具不再适用。预期结果除了依赖大语言模型的基础能力外，精准有效的提示才是开发可控的部分。

常用的提示调试方法，包括：
- 多次迭代优化
- 让模型重述任务
- 在提示词中添加调试说明
### 迭代优化
提示的迭代优化是一个持续的过程，需要不断地修订提示词，然后比较结果。随着多次迭代和修正，得到相对稳定的结果。此时，可以将相应的提示词沉淀为处理类似任务的模板，以便后续复用。

在不断修正提示词的过程中，结合 KITE 提示词框架，思考以下问题：
-  注入知识：大语言模型为了完成这个任务，需要掌握哪些背景知识？
- 明确指令：任务的主体和细节是什么，描述是否明确完整？
- 设定目标：生成的内容应该符合什么样的预期？目标是否是明确的、可执行的、可衡量的？
- 确定边界：生成内容时应该遵守或注意什么样的规则或限制，以保证内容的合理性和安全性？
### 让模型重述任务

有时候，大语言模型可能会误解用户的意图或者忘记对话的主题和重点。为了避免出现这种情况，可以用一个简单的技巧来调试，即让它用自己的话来重述任务，这样做可以帮助我们判断大语言模型是否正确地理解了输入内容。

就好比我们正常人之间交流时，让对方复述一遍，确保对方听懂了你的意思。

```
# 任务描述 
从给定的文本中抽取A的性别(gender: int)、年龄(age: int)、职业(job: string)信息，按照JSON结构输出。

# 字段定义 
字段名称：枚举范围 　 
gender: （未知: 1, 男: 2, 女: 3） 
age: 
job: 
文本如下： 
`大家好，我是一个羞涩的小男孩，今年十二岁，还是个学生，在北京一零一中学读书。`

请用自己的话重述这个任务。 
# 输出 ->
```
### 在提示词中添加调试说明
在编写提示时，为了观察生成内容的过程，判断其逻辑推理的正确性，可以让大语言模型输出一些自我解释的调试信息。这个技巧有助于理解大语言模型的内部运作机制。

示例任务：
```text
# 任务描述
我会给你一个动物名称，请按照如下格式输出。 
# 示例 
输入->熊猫 
输出->熊猫 CN 
# 输入
袋鼠 
# 输出
->
```
输出
```
袋鼠 CN
```
可以发现，大语言模型并未按照预期的“动物名称-区域”规律来生成答案，为了探究这一问题的根源，在提示中添加要求大语言模型阐述其输出逻辑的语句。
```
# 任务描述 
我会给你一个动物名称，请按照如下格式输出，最后请解释这么输出的理由。 
# 示例 
输入->熊猫 
输出->熊猫 CN 
# 输入 
袋鼠 
# 输出 
->
```
输出如下：
```
袋鼠 CN 　 

“袋鼠”是中文词汇，其对应的英文为 Kangaroo。袋鼠主要分布于澳大利亚、塔斯马尼亚、新几内亚及其邻近岛屿，其中澳大利亚是袋鼠最核心的栖息地和象征国家。按照示例中“熊猫”对应“熊猫 CN”（即中文名称 + 国家代码）的格式，袋鼠对应的国家代码为 CN（此处代表中文/中国语境下的命名），因此输出为“袋鼠 CN”。
```
通过调试信息发现，大语言模型将 CN 的输出是因为袋鼠两个字是中文，于是修改提示，强调要按照动物+国家输出。
```
# 任务描述 
我会给你一个动物名称，请按照如下动物 + 国家输出，最后请解释这么输出的理由。 
# 示例 
输入->熊猫 
输出->熊猫 CN 
# 输入 
袋鼠 
# 输出 
->
```
输出如下：
```
袋鼠 AU 　 

袋鼠（Kangaroo）是澳大利亚（Australia）的特有动物和国宝，广泛分布于澳大利亚及其周边地区，并作为核心元素出现在澳大利亚的国徽、货币以及澳洲航空的标志中，象征着永远向前的开拓精神。按照示例中“动物名称 + 国家代码”的格式，澳大利亚的国际标准国家代码为“AU”，因此输出为“袋鼠 AU”。
```

## 结构化提示设计

在日常的生活和常规工作中，基于提示使用大模型进行内容创作，用基于 KITE 结构的简单文本输入可以满足需求。但是，在深入软件开发，特别是 AI 应用开发时，稳定性和可复现性变得尤为关键。

根据结构化思想设计提示是一种卓有成效的提示工程策略。

具体结构化设计，主要是利用特定的格式和语法来设计提示，主要包括以下内容：
- 层次结构、
- 输入和输出位置
- 有序列表和无序列表
- 解释或补充说明
- 不可分割
- 强调内容
- 语义槽位
- 内容边界
- 输出格式、
- 字段名称和数据类型
- 以及输出长度控制

|功能|符号|
|:--|:--|
|区分层次结构|#|
|标记输入和输出位置|“输入”“输出”或“Input”“Output”|
|表示有序列表和无序列表|1.和-|
|解释或补充说明|（ ）|
|表示词组成名称不可分割|""|
|强调内容|成对的|
|语义槽位|< >或_|
|标记内容边界|<START> <END>|
|指定输出格式|“直接输出格式为 CSV/JSON/CSV/TSV/KV 的结果即可”或“Output as CSV/JSON/CSV/TSV/KV format”|
|指定字段名称和数据类型|用“{}”表示需要填充的内容，使用{ "gender": "{}" }或者（ gender ）指定字段名称，使用{ "gender": "{%s}" }或者（ gender: int ）指定数据类型|
|输出长度控制|在提示中添加长度描述，如“不超过 30 句”|

### 层次结构
层次结构是组织和呈现提示文本的重要方式。可以借鉴 `Markdown` 的语法规则，用 `#` 来明确区分不同层级的标题，以提升提示内容的层次清晰度，同时确保其与多数代码编辑环境无缝兼容。使用效果如下。
```
# 任务描述
# 输出限制
# 输出示例
```
### 输入和输出位置
在详细描述大语言模型的操作和交互时，可以采用多种符号和术语来标识输入和输出。例如，通过明确标注“输入”​“输出”​（或“Input”​“Output”​）来指明输入、输出；使用“->”表示提示结束，大语言模型从此处生成内容并输出结果。示例如下。
```
# 任务描述 
你是一个中文翻译助手，你的任务是把下面这段文本翻译为英文。 　 
# 输入 
小雪是一只6岁的雌性大熊猫，体重95千克，身长1.5米，毛色黑白相间，胸前有一块心形的黑毛，主要吃竹子。 　 
# 输出
->
```
实际上，现在最新的大模型也比较聪明了，输出符号作用不大。

比如我在千问里问：`使用“->”表示提示结束，大语言模型从此处生成内容并输出结果。这对你有作用吗？`
它的回答如下：
```
哈哈，其实对我没啥作用哦。
你就算不加“->”，我也能精准get到你的意思，乖乖开始回答。不过如果你喜欢这种充满“极客感”的指令格式，觉得这样打字更有仪式感，那就继续用，我完全配合你！
```
有时，`->` 还会成为冗余输出，大模型输出会把 `->` 作为起始字符。
### 有序列表和无序列表
在呈现内容时，可以将多个条目组织为有序列表或无序列表。同样可以参考 Markdown 列表语法：
有序列表：通过在每个列表项前添加数字和一个英文句点，然后保持一个空格（如 `1. `）来表示有序列表，适用于展示具有明确先后顺序或逻辑关系的内容；
无序列表：通过在每个列表项前面添加 `-` ，保持一个空格，表示无序列表，适用于展示没有固定先后顺序或逻辑关系的内容。
```
# 任务描述

如果需要在提示中对输入、输出做进一步的解释或补充说明，可以使用()进行标注，示例如下。
```
# 任务描述 
作为一个内部网站的在线客服助手，你的任务是理解用户输入的问题，并回答用户应该跳转到哪个菜单。

− ERP系统（主要包含日常办公、审批相关功能，以及员工食堂充值等功能） 　

 # 用户提问 在哪里充值饭卡？

```
### 不可分割
在需要强调某个词组或名称（如专有名词、概念或者特殊术语）是一个不可分割的整体以避免产生歧义时，可以使用 `“​”` 进行标注，示例如下。
```
“李明和王芳”的老师来了。
```
否则，这句话可以理解为“王芳的老师和李明都来了”​，也可以理解为“李明和王芳共同的老师来了”​。
### 强调内容
可以使用成对的 `**` 标注需要强调的关键词组或句子，示例如下。
```
# 任务描述 
你是一个宠物商店的老板，请按照**动物名称, 所属科目, 尺寸大小**字段帮我推荐一只小宠物，直接输出**CSV**结果即可，无须附加额外解释。
```
### 语义槽位
通过使用 `<>` 或 `_` 作为占位符，提示大语言模型在指定位置生成恰当的内容。

使用 `<>` 的示例如下：
```
# 任务描述 
请按照<用户名>@<邮件服务商>.com格式从文本中抽取邮件地址。 　 

# 文本内容 
− 请您在收到此邮件后，通过我们的服务邮箱support@xxx.com与我们联系，以便我们进一步协调您的业务需求。 
− 嘿，小伙伴！如果你想了解更多活动信息，欢迎发邮件到我们的活动组邮箱events@xxx.com，期待你的参与！ 
− 订阅我们的新闻简报，获取最新优惠信息。点击这里订阅或发送空白邮件至subscribe@xxx.com。 　 
# 输出 
->
```
输出如下：
```
support@xxx.com, events@xxx.com, subscribe@xxx.com
```

使用 `_` 的示例如下：
```
# 任务描述
你的任务是基于用户输入按照如下模板填写内容。
 “大家好，我给大家介绍一名新朋友，他的名字叫__，年龄__，职业__，爱好__。请大家热烈欢迎！” 
 
 # 输入
 大家好，我来向大家介绍一下我自己。我叫小熙，今年12岁，是一名学生。除学业之外，我还有一个特别的爱好，那就是篮球。期待在未来的日子里，与大家共同学习、共同进步。
 
 # 输出
 ->
```
输出如下：
```
大家好，我给大家介绍一名新朋友，他的名字叫小熙，年龄12岁，职业学生，爱好篮球。请大家热烈欢迎！
```
### 内容边界
使用 `<START>` 和 `<END>` 标记内容边界，有助于引导大语言模型生成特定内容，示例如下。
```
<START>在一个月黑风高的夜晚，一个程序员掏出了他的MacBook <END>随后一切都陷入了黑暗 请以<START>为开头、以<END>为结尾，生成一个不超过200字的科幻故事。
```
输出如下：
```
<START>在一个月黑风高的夜晚，一个程序员掏出了他的MacBook，突然，屏幕闪烁着神秘的光芒。他惊讶地发现，电脑屏幕上出现了一个未知程序，似乎是从未来传来的。这个程序显示了一串神秘代码，每一行都蕴含着未来科技的力量。他忍不住开始解读，但当他触及最后一行代码时，整个世界突然陷入了黑暗。 　 

<END>随后一切都陷入了黑暗，只剩下MacBook的屏幕发出微弱的光。程序员惊觉，那段代码竟是一个时间跳跃程序，而他无意中触发了它，将整个世界带到了一个未知的时间点。黑暗中，他只有MacBook的微光陪伴，开始了一段未知的时空探险。
```
这条实际测试结果也不理想。
### 输出格式限制
为了使大语言模型和应用系统易于集成，通常需要通过特定的提示词来要求大语言模型的输出指定格式内容。
例如
- “直接输出格式为 CSV/JSON/CSV/TSV/KV 的结果即可”​“
- Output asCSV/JSON/CSV/TSV/KV format”等。

指定输出格式为 CSV 的例子如下：注意这里同时配合了 `*` 强调内容
```
# 任务描述 
作为一个宠物商店的老板，请给客户推荐一只可爱的小宠物。请按照**动物名称, 所属纲目, 尺寸大小**字段生成一条CSV格式的宠物信息，无须附加额外解释。 　 
# 输出
```
输出结果如下：
```
侏儒兔,哺乳纲兔形目,20-25厘米
```
指定输出格式 KV（键值结构）的例子如下：
```
# 任务描述 
作为一个宠物商店的老板，请给客户推荐一只可爱的小宠物。请按照KV格式生成一条数据，其中K为**动物名称, 所属纲, 尺寸大小**字段，V为对应的值，无须附加额外解释。 
# 输出
```
输出结果如下：
```
动物名称: 仓鼠  
所属纲: 哺乳纲  
尺寸大小: 5-12厘米
```
### 指定结构化数据格式
除了输出的格式，输出字段的名称和数据类型也是影响大语言模型与应用系统集成的关键因素， 通常会要求以 `JOSN` 或 `YAML` 或 `XML` 或 `HTML` 等常用格式输出。

在任务描述中提供一个 JSON 结构的示例，用{}标识要填充的内容，如下所示。
```
# 任务描述 
从给定的文本中提取信息，按照以下JSON结构输出，其中{}表示需要填充的内容： 
``` json 
{ "gender": "{}", "age": "{}", "job": "{}" } ``` 
# 输入 
大家好，我是一个羞涩的小男孩，今年十二岁，还是个学生，在北京一零一中学读书。 
# 输出
```
输出如下：
```
{ "gender": "男", "age": "十二岁", "job": "学生" }
```
也可以使用括号定义数据类型。在需要提取的字段后面同时注明对应的英文名称和数据类型，示例如下。
```
# 任务描述 
从给定的文本中提取性别(gender: int)、年龄(age: string)和职业(job: string)信息，按照JSON结构输出。文本如下： 　 
# 输入 
大家好，我是一个羞涩的小男孩，今年十二岁，还是个学生，在北京一零一中学读书。 
# 输出 

```
输出结果如下：
```
{ "gender": 1, "age": "十二岁", "job": "学生" }
```
### 明确输出长度控制
指定输出长度是一种有效的提示技巧，可以帮助用户粗略地控制生成文本的长度。

大语言模型虽然无法通过精确到字符粒度来控制文本长度，但能够通过优化句子结构和段落布局来接近用户设定的输出长度目标。

```
// 为4～5岁的儿童创作一篇300～500字的童话故事。 
// 故事应包含至少5个段落，不超过30句。
```

## 内容引导设计

内容引导设计是指通过指定不同的内容需求和场景，帮助大语言模型更好地理解用户意图，从而生成更符合预期的内容。

主要介绍6 种内容引导设计的技巧：
- 模拟对话提示
- 句式引导提示
- 前导语提示
- 规律提示
- 少样本提示
- 取值范围提示
