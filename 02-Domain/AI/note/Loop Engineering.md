Loop 是 AI 工程的新范式，标志着工程师从"写代码"到"设计系统"的认知跃迁。

它由五大核心模块构成：
- 自动化调度（/loop、/goal 等指令实现循环）
- 工作树隔离（解决多 Agent 协作冲突）
- Skill（用 SKILL.md 固化项目意图）
- 插件连接器（通过 MCP 协议对接外部系统）
- 子 Agent（写代码与审查分离）
- 关键补充是记忆机制——模型会遗忘，但仓库不会。

Loop 不替代工程师，而是将人的焦点从 Prompt 移至系统设计：验证责任、防止理解债、避免认知放弃仍是人类核心价值。

Claude Fable 5 在此架构下展现出更强的持续学习能力，但 Loop 效能终究取决于设计者的工程深度。
